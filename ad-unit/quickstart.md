# ⚡ 퀵스타트: 첫 광고 유닛 만들기

> ⏱️ 소요 시간: 약 5분

## 시작 전 준비

- 앱 이름
- 패키지명(Android) / 번들ID(iOS) / 도메인(Web)

## 단계별 가이드

[▶ 영상 보기](./assets/ad-unit-create.mp4)

### 1단계: 앱 등록

광고 유닛을 생성하려면 먼저 앱을 등록해야 합니다.

1. 콘솔 접속 → **광고 유닛** 메뉴 선택
2. **앱 추가** 클릭
3. 앱 정보 입력
   - **앱 이름**: 스토어 검색 또는 직접 입력
   - **플랫폼**: Android / iOS / Web
   - **패키지명**(Android) / **번들ID**(iOS) / **도메인**(Web)
4. **저장** → 앱 ID 발급

> 💡 Google Play, App Store에 등록된 앱은 검색으로 빠르게 추가할 수 있습니다.

### 2단계: 광고 유닛 생성

1. 등록된 앱 선택
2. **광고 유닛 추가** 클릭
3. 광고 유닛 정보 입력
   - **포맷**: 배너, 전면, 보상형, 네이티브, 팝업, 스플래시 중 선택
   - **사이즈**: 포맷에 따라 선택 가능한 사이즈가 다름
   - **광고 유닛명**: 관리용 이름 (예: 홈_상단_배너)
4. **저장** → 광고 유닛 ID 발급

> 💡 [광고 상품 생성](../ad-product/quickstart.md) 화면에서도 광고 유닛을 함께 만들 수 있습니다.

## 완료

생성이 완료되면 앱 ID와 광고 유닛 ID가 발급됩니다.

| 항목 | 확인 위치 | 용도 |
|------|----------|------|
| **앱 ID** | 설정 > 앱 | SDK 초기화 시 사용 |
| **광고 유닛 ID** | 광고 유닛 목록 | 광고 요청 시 사용 |

발급된 ID를 개발자에게 전달하세요. 개발자는 SDK를 연동하여 해당 광고 유닛에 광고를 노출합니다.

## FAQ

**Q. 광고 유닛과 광고 상품의 차이는 무엇인가요?**

광고 유닛은 실제 광고가 노출되는 개별 영역이고, 광고 상품은 광고 유닛을 묶어 판매하는 단위입니다. 광고주는 광고 상품을 구매하고, 해당 상품에 포함된 광고 유닛에 광고가 노출됩니다.

**Q. 백필 광고를 활용하려면 어떤 사이즈로 광고 유닛을 만들어야 하나요?**

백필 광고의 Fill Rate를 높이려면 글로벌 광고 네트워크에서 널리 사용되는 표준 사이즈를 권장합니다.

| 사이즈 | 설명 |
|--------|------|
| 320x50 | 모바일 기본 배너, 가장 높은 Fill Rate |
| 320x100 | 대형 배너 |
| 300x250 | 미디엄 렉탱글, 높은 eCPM |

**Q. 광고 유닛을 생성한 후 수정해도 되나요?**

가능하면 새 광고 유닛을 생성하는 것을 권장합니다. 이미 배포된 앱에서 해당 광고 유닛을 사용 중일 수 있으므로, 수정 시 예기치 않은 문제가 발생할 수 있습니다.
