# 문맥 타겟팅 만들기

문맥 타겟팅은 콘텐츠 속성을 기반으로 특정 페이지나 콘텐츠에 광고를 노출하는 기능입니다.

이 가이드에서는 **로컬 액티비티 앱**을 예시로 문맥 타겟팅을 만드는 방법을 설명합니다. 사용자가 보고 있는 액티비티 카테고리를 기반으로 타겟팅 카테고리를 구성합니다.

---

## 컨텍스트란?

**컨텍스트**는 사용자가 현재 보고 있는 페이지의 맥락 정보입니다. 상품 카테고리, 기사 분류, 검색 키워드 등 페이지를 구분할 수 있는 모든 데이터가 컨텍스트가 될 수 있습니다.

컨텍스트는 광고 요청 시 SDK를 통해 전달되며, 전달된 컨텍스트를 조합하여 타겟팅 카테고리를 만듭니다.

> 💡 컨텍스트 데이터의 유효기간은 28일입니다.

---

## 컨텍스트 ID 전달 방식

컨텍스트 ID는 매체사가 직접 정의합니다. 이미 서비스에서 사용 중인 콘텐츠 분류 체계가 있다면 그대로 활용할 수 있습니다. SDK에서 광고 요청 시 컨텍스트 ID를 함께 전달하면, 해당 맥락에 맞는 광고가 노출됩니다.

> ⚠️ 컨텍스트 ID는 대소문자를 구분합니다. SDK에서 전달하는 값과 동일하게 입력하세요.

### 예시: 로컬 액티비티 앱

로컬 액티비티 앱에서 사용자가 보고 있는 액티비티 상세 페이지의 카테고리를 컨텍스트로 전달한다고 가정합니다.

| 액티비티 페이지 | 컨텍스트 ID |
|-----------------|-------------|
| 북한산 등산 모임 | hiking |
| 서핑 레슨 | surfing |
| 한강 카약 체험 | outdoor |
| 글램핑 파티 | camping |
| 이탈리안 쿠킹 클래스 | cooking |
| 전통주 시음회 | tasting |
| 제주 맛집 투어 | food |
| 도자기 공방 체험 | craft |
| 현대미술 전시 투어 | exhibition |
| 뮤지컬 관람 | culture |
| 플라워 아트 클래스 | art |
| 요가 클래스 | yoga |
| 명상 워크숍 | meditation |
| 스파 데이 | spa |

## 문맥 타겟팅 생성하기

[▶ 영상 보기](./assets/targeting-context-category.mp4)

### 1단계: 타겟팅 생성

1. 좌측 메뉴에서 **타겟팅**을 클릭합니다.
2. 문맥 타겟팅 섹션에서 **문맥 타겟팅 생성** 버튼을 클릭합니다.

### 2단계: 타겟팅 정보 입력

**타겟팅 이름**

광고주에게 표시될 타겟팅 이름을 입력합니다.

**할증률**

이 타겟팅을 선택했을 때 적용할 할증률을 설정합니다. (0% ~ 200%)

### 3단계: 카테고리 추가

1. **+ 커스텀 카테고리 추가** 버튼을 클릭합니다.
2. 카테고리 이름을 입력합니다.
3. 조건을 설정합니다.
   - 연산자 선택 (같음, 포함 등)
   - 컨텍스트 ID 입력
4. 필요시 **조건 추가**로 여러 컨텍스트 ID를 조합합니다.
5. **추가** 버튼을 클릭합니다.

**예시: 로컬 액티비티 앱 카테고리 구성**

컨텍스트 ID를 조합하여 "액티비티 카테고리" 타겟팅을 만듭니다.

| 카테고리 | 조건 |
|----------|------|
| 아웃도어 | context is "outdoor", "camping", "hiking", "surfing" |
| 푸드 | context is "food", "cooking", "tasting" |
| 문화 | context is "culture", "exhibition", "concert" |
| 예술 | context is "art", "craft", "design" |
| 웰니스 | context is "wellness", "yoga", "meditation", "spa" |

### 4단계: 저장

**타겟팅 생성** 버튼을 클릭하여 저장합니다.

## 카테고리 그룹 설정

[▶ 영상 보기](./assets/targeting-context-category-group.mp4)

여러 카테고리를 하나로 묶어 광고주에게 제공할 수 있습니다.

1. 생성된 타겟팅 카드를 클릭합니다.
2. **카테고리 그룹 설정** 버튼을 클릭합니다.
3. 그룹 이름을 입력합니다.
4. 포함할 카테고리를 선택합니다.
5. **저장** 버튼을 클릭합니다.

> ⚠️ 그룹으로 묶인 카테고리는 개별 선택이 불가능해집니다.

**예시: 문화/예술 그룹 만들기**

문화와 예술 카테고리를 묶어 "문화/예술" 그룹을 만듭니다.

| 그룹 | 포함 카테고리 |
|------|---------------|
| 문화/예술 | 문화, 예술 |

그룹 설정 후 광고주에게 표시되는 옵션:

- 아웃도어
- 푸드
- 웰니스
- **문화/예술** (그룹)

## FAQ

**Q. 컨텍스트 ID는 어떻게 설계해야 하나요?**

A. 이미 서비스에서 사용 중인 콘텐츠 분류 체계를 그대로 활용하는 것이 좋습니다. 새로 만들 필요 없이 기존 카테고리 코드나 태그를 컨텍스트 ID로 전달하면 됩니다.

**Q. 문맥 타겟팅은 로그인하지 않은 사용자에게도 적용되나요?**

A. 네, 문맥 타겟팅은 사용자가 현재 보고 있는 페이지의 콘텐츠를 기준으로 합니다. 사용자 정보가 필요 없으므로 비로그인 사용자에게도 동일하게 적용됩니다.

**Q. 문맥 타겟팅은 어떤 서비스에 적합한가요?**

A. 콘텐츠 분류가 명확한 서비스에 효과적입니다. 뉴스 앱의 기사 카테고리, 커머스의 상품 카테고리, 부동산 앱의 매물 유형처럼 페이지별 맥락이 뚜렷하면 광고주에게 가치 있는 타겟팅을 제공할 수 있습니다.
