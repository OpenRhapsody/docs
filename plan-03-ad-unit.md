# 광고 유닛 설정

## ⚡ 퀵스타트: 첫 유닛 만들기
- 🎬 데모 영상: 앱 등록부터 유닛 ID 발급까지 전체 과정
- ⏱️ 소요 시간: 약 5분
- 📋 시작 전 준비:
  - 앱 이름
  - 패키지명(Android) / 번들ID(iOS) / 도메인(Web)
- ✅ 완료 후: 유닛 ID를 개발자에게 전달
- 👉 다음 단계: 광고 상품 만들기

## 광고 유닛이란?
- 유닛 개념 (실제 광고 노출 영역)
- 상품과 유닛의 관계

## 유닛 포맷

### 배너 (Banner)
- 설명: 고정 영역에 표시되는 직사각형 광고
- 사이즈: 320x50, 320x100, 300x250, 375x80, 16:9(동영상), 9:16(동영상), 커스텀
- 옵션:
  - 캐러셀 활성화
  - 캐러셀 인디케이터 표시
  - 캐러셀 롤링 간격
  - 캐러셀 최대 개수

### 전면 (Interstitial)
- 설명: 화면 전체를 덮는 광고
- 사이즈: 320x480, 360x360, 16:9(동영상), 9:16(동영상)
- 옵션:
  - 닫기 버튼 지연 시간

### 보상형 (Rewarded)
- 설명: 시청 완료 시 보상 제공, 동영상 전용
- 사이즈: 9:16 (동영상)
- 옵션:
  - 보상 콜백 설정

### 네이티브 (Native)
- 설명: 콘텐츠와 자연스럽게 어우러지는 광고
- 사이즈: 360x360, 1:1, 16:9(동영상), 9:16(동영상), 커스텀, 텍스트 전용
- 옵션:
  - 광고주 프로필 표시
  - CTA 버튼 표시
  - 텍스트 항목 설정 (라벨, 최대 길이, 필수 여부)

### 팝업 (Popup)
- 설명: 화면 일부에 팝업으로 표시 (하단/중앙)
- 사이즈: 360x240(하단), 360x450(하단), 360x360(중앙), 커스텀
- 옵션:
  - 닫기 버튼 위치 (내부 상단/하단, 외부 상단/하단)
  - 딤 영역 탭하여 닫기
  - 딤 투명도
  - 오늘 하루 유닛 보지 않기
  - 오늘 하루 캠페인 보지 않기
  - 템플릿 (오늘 하루 보지 않기 / 성과형)

### 스플래시 (Splash)
- 설명: 앱 실행 시 표시
- 사이즈: 360x270
- 옵션:
  - 닫기 시간 설정

## 동영상 광고 유닛
- 🎬 데모 영상: 동영상 유닛 생성 및 사이즈 설정
- 설명: 동영상 크리에이티브를 지원하는 광고 포맷
- 지원 포맷: 배너, 전면, 보상형, 네이티브
- 사이즈: 16:9 (가로형), 9:16 (세로형)
- 특징:
  - 자동 재생 / 사용자 재생 선택
  - 음소거 기본 설정
  - 보상형은 동영상 전용

## 다크 모드 지원하기
- 🎬 데모 영상: 다크 모드 옵션 활성화 및 소재 등록
- 지원 포맷: 배너, 팝업, 스플래시, 네이티브 (동영상 미지원)
- 설정 방법: 유닛 생성/수정 시 다크 모드 옵션 스위치 활성화
- 주의사항 (⚠️)
  - 진행 중 캠페인: 다크 모드 활성화 시 라이트 모드 소재가 기본 노출
  - 심사: 라이트/다크 모드 소재 동시 심사, 하나라도 위반 시 거절

## 유닛 생성하기
- 앱 등록 (유닛 생성 전 필수)
  - 지원 플랫폼: Android, iOS, Web
  - 앱 정보 입력 (앱 이름, 패키지명/번들ID/도메인)
  - 앱 ID 확인 (개발자에게 전달)
- 유닛 생성 단계
  - 앱 선택
  - 포맷 및 사이즈 선택
  - 유닛명 입력
  - 유닛 ID 확인 (개발자에게 전달)
- 공통 옵션
  - 배경색 설정
  - 다크 테마 지원
  - 크리에이티브 미리보기

## 유닛 수정하기
- 수정 가능한 항목
  - 유닛명
  - 포맷·크기 (판매 중인 상품에 미포함 시)
  - 옵션 설정
- 수정 불가능한 항목 (⚠️ 주의)
  - 포맷·크기: 판매 중인 상품에 포함된 유닛은 수정 불가
  - 연결된 앱: 변경 불가
- 캠페인 반영 시점 (⚠️ 중요)
  - 팝업: 운영 중인 캠페인에 즉시 적용
  - 팝업 외 (배너, 전면, 보상형, 네이티브, 스플래시): 새 캠페인부터 적용
- 유닛 세트 충돌 (⚠️ 주의)
  - 수정으로 유닛 세트 조건 불일치 시 상품이 캠페인 예약에 노출 안됨
  - 수정 후 관련 상품 상태 확인 필수

## 유닛 삭제하기
- 삭제 방식
  - 소프트 삭제 (보관 처리)
  - 즉시 삭제되지 않고 archived 상태로 전환
- 삭제 시 자동 처리
  - 연결된 상품의 유닛 세트에서 자동 제거
  - 유닛 세트가 비게 되면 해당 상품 판매 불가
- 삭제 후 영향 (⚠️ 주의)
  - 해당 유닛의 과거 리포트 데이터는 유지
  - SDK에서 해당 유닛 ID로 광고 요청 시 에러 반환
  - 삭제된 유닛 ID는 재사용 불가

## 백필 광고로 추가 수익화하기
- 🎬 데모 영상: 백필 광고 활성화 및 상태 확인
- 백필이란?
  - 직광고가 없을 때 여러 광고 소스를 자동 연동하여 빈 인벤토리 채움
  - 비딩·워터폴 최적화로 eCPM과 Fill Rate 동시 향상
- 활성화 방법
  - 광고 유닛 목록에서 'Backfill 광고 켜기' 클릭
  - 웹 SDK 연동 여부 선택
  - 상태가 '처리중'으로 변경, 연결 완료 시 자동 수익화 시작
- 주의사항 (⚠️)
  - 반영 시간: 일시정지/재시작 시 최대 30분 소요
  - 네이티브 유닛: 추가 텍스트 항목 노출 안됨, Headline 또는 이미지 필수
- 성과 확인
  - 리포트 메뉴의 Backfill 탭에서 조회
