# 캠페인 상세 정보

캠페인 상세 화면에서 확인하고 설정할 수 있는 항목들을 안내합니다.

---

## 캠페인 현황

캠페인을 클릭하면 상세 화면이 열립니다. 캠페인의 설정 정보와 운영 현황을 한눈에 확인할 수 있습니다.

### 캠페인 설정

| 항목 | 설명 |
|------|------|
| **캠페인 ID** | 자동 부여되는 고유 식별자. 클릭하여 복사 가능 |
| **광고 상품** | 상품명, 할인율, 예상 CTR |
| **과금 유형** | CPM, CPC 등 과금 방식과 단가 |
| **예산** | 캠페인 총 예산 |
| **기간** | 시작일 ~ 종료일 (타임존 포함) |
| **국가** | 광고 노출 국가 |
| **타겟팅** | 포함/제외 카테고리, 타겟팅별 추가 요금율 |

### 광고주 정보

| 항목 | 설명 |
|------|------|
| **광고주** | 광고주명, 웹사이트 URL |
| **계약 담당자** | 담당자 이름, 이메일 |
| **결제 수단** | 카드 결제(선불) 또는 수동 결제(후불), 영수증 링크 |
| **청구 정보** | 사업자명, 사업자등록번호, 청구 이메일 |

### 예산 소진

진행중 또는 일시정지 상태의 캠페인에서 표시됩니다.

- **소진 금액**: 현재까지 소진된 예산과 총 예산
- **진행률**: 원형 프로그레스 바로 실제 소진(검은색)과 예상 소진(회색) 비교
- **상태 해석**: 예상 대비 실제 소진 상태 (예: "예상보다 빠르게 소진 중")

### 성과

| 지표 | 설명 |
|------|------|
| **전체 노출수** | 누적 노출 횟수. CPM 캠페인은 목표 대비 달성률 표시 |
| **일일 노출수** | 오늘의 노출수와 일 할당량 대비 달성률 |
| **전체 클릭수** | 누적 클릭 횟수. CPC 캠페인은 목표 대비 달성률 표시 |
| **일일 클릭수** | 오늘의 클릭수와 일 할당량 대비 달성률 |

**성과 리포트** 버튼을 클릭하면 일별 상세 성과를 확인할 수 있습니다. 취소되거나 종료된 캠페인은 **정산서** 버튼도 표시됩니다.

---

## 광고 소재

캠페인에 등록된 소재 목록과 각 소재의 성과를 확인할 수 있습니다. 심사 대기 중인 소재가 있으면 상단에 별도로 표시되며, 클릭하여 바로 심사할 수 있습니다.

> ⚠️ 필수 소재가 등록되지 않은 경우 경고 배너가 표시됩니다.

### 소재 목록

| 항목 | 설명 |
|------|------|
| **상태** | 심사중, 승인됨, 거절됨, 일시정지 |
| **소재명** | 소재 이름 |
| **유형** | 광고 유닛 세트 |
| **포맷** | 형식과 크기 (예: Display 300x250) |
| **노출수** | 소재별 노출 횟수 |
| **클릭수** | 소재별 클릭 횟수 |
| **CTR** | 소재별 클릭률 |

### 소재 미리보기

소재를 클릭하면 미리보기 화면이 열립니다. 이미지는 실제 크기로, 동영상은 재생 미리보기로 확인할 수 있습니다. 라이트/다크 모드 전환도 가능합니다.

> 💡 동영상 소재가 인코딩 중이면 완료 후 미리보기가 가능합니다.

## 노출 빈도 제한

동일 사용자에게 같은 캠페인의 광고가 반복 노출되는 것을 제한합니다. 예를 들어 빈도를 50으로 설정하면, 한 사용자에게 해당 캠페인의 광고가 하루에 최대 50회까지만 노출됩니다.

예약됨, 진행중, 일시정지 상태의 캠페인에서 설정할 수 있습니다.

### 왜 필요한가요?

같은 광고를 너무 자주 보면 사용자는 피로감을 느끼고, 심하면 브랜드에 부정적인 인식을 갖게 됩니다. 이를 **광고 피로도(Ad Fatigue)**라고 합니다. 빈도 제한은 이를 방지하면서도 충분한 노출을 확보하는 균형점을 찾는 설정입니다.

### 활용 가이드

| 상황 | 권장 빈도 | 이유 |
|------|----------|------|
| **브랜딩 캠페인** | 낮게 (3~10회) | 브랜드 인지도는 적은 노출로도 형성되며, 과다 노출은 역효과 |
| **프로모션/이벤트** | 중간 (10~30회) | 긴급성 있는 메시지는 반복 노출이 효과적 |
| **리타겟팅** | 높게 (30~50회) | 이미 관심을 보인 사용자에게는 더 많은 노출이 전환에 유리 |

> 💡 빈도가 너무 낮으면 예산을 다 소진하지 못할 수 있고, 너무 높으면 소수 사용자에게 광고가 집중됩니다. 캠페인 목표와 DAU를 고려해 설정하세요.

### 설정 방법

1. 캠페인 상세 화면에서 **노출 빈도 제한** 항목을 클릭합니다.
2. 원하는 빈도를 입력합니다. (기본값: 50회/일)
3. **저장** 버튼을 클릭합니다.

## 일 할당량

캠페인의 일일 노출(CPM) 또는 클릭(CPC) 목표를 설정합니다. 예약됨, 진행중, 일시정지 상태의 캠페인에서 설정 가능합니다.

### 왜 필요한가요?

광고 캠페인은 보통 특정 기간 동안 목표량을 달성해야 합니다. 일 할당량은 이 목표를 기간 내에 균등하게 달성하도록 **페이싱(Pacing)**을 조절하는 설정입니다.

페이싱이 없으면 트래픽이 많은 초반에 예산이 빠르게 소진되어, 후반에는 광고가 노출되지 않을 수 있습니다. 반대로 너무 보수적이면 기간이 끝났는데 목표를 달성하지 못할 수 있습니다.

### 자동 vs 수동

| 모드 | 동작 | 적합한 상황 |
|------|------|------------|
| **자동** | 매일 자정에 남은 목표 기준으로 재계산하며, 목표 달성률을 높이기 위해 자동 최적화 | 대부분의 캠페인 |
| **수동** | 지정한 값으로 일 할당량 고정 | 특정 요일에 집중 노출하거나, 이벤트 일정에 맞춰 조절할 때 |

### 설정 방법

1. 캠페인 상세 화면에서 **일 노출 할당량** (CPM) 또는 **일 클릭 할당량** (CPC) 항목을 클릭합니다.
2. **자동** 또는 **수동**을 선택합니다.
3. 수동 선택 시 원하는 일일 할당량을 입력합니다.
4. **저장** 버튼을 클릭합니다.

> 💡 수동 설정 시에도 총 목표량을 초과하는 값은 입력할 수 없습니다.

## 테스트 기기에서 소재 미리보기

실제 기기에서 광고가 어떻게 보이는지 확인할 수 있습니다. 테스트 기기를 등록하면 해당 기기에서 소재를 미리 볼 수 있습니다.

### 테스트 기기 등록

1. 우측 상단 프로필 메뉴에서 **테스트 기기**를 클릭합니다.
2. **기기 추가** 버튼을 클릭합니다.
3. 다음 정보를 입력합니다.
   - **기기 이름**: 콘솔에서 구분하기 위한 이름 (2~30자)
   - **ADID**: 기기의 광고 식별자 (UUID 형식)
4. **추가** 버튼을 클릭합니다.

### ADID 확인 방법

ADID(Advertising ID)는 기기별로 부여되는 광고 식별자입니다.

**Android**

1. **설정** 앱을 엽니다.
2. **보안 및 개인정보 보호** > **기타 개인정보 설정**으로 이동합니다.
3. **광고**를 선택합니다.
4. **광고 ID**를 확인합니다.

[AAID - Find your Google Advert](https://play.google.com/store/apps/details?id=com.kyumpany.aaid) 앱을 설치하면 ADID를 클립보드에 복사할 수 있습니다.

**iOS (IDFA)**

1. **설정** 앱을 엽니다.
2. **개인정보 보호 및 보안** > **추적**으로 이동합니다.
3. 앱의 추적 권한이 활성화되어 있는지 확인합니다.

[Get My IDFA](https://apps.apple.com/kr/app/get-my-idfa/id1580368827) 앱을 설치하고 추적을 허용하면 IDFA를 클립보드에 복사할 수 있습니다.

> ⚠️ iOS는 ATT(App Tracking Transparency) 정책에 따라 사용자의 추적 동의가 필요합니다. 추적을 거부하면 광고 식별자에 접근할 수 없습니다.

### 미리보기 실행

1. 소재 미리보기 화면에서 **내 기기에서 보기** 버튼을 클릭합니다.
2. 등록된 테스트 기기를 선택합니다.
3. **미리보기** 버튼을 클릭합니다.
4. 선택한 기기에서 앱을 실행하면 해당 소재가 노출됩니다.

### 지원 환경

미리보기 기능은 기기의 광고 식별자(ADID/IDFA)를 사용하여 동작합니다. 다음 SDK 버전 이상이 필요합니다.

| 플랫폼 | 최소 버전 |
|--------|----------|
| **Android** | 1.3.1 |
| **iOS** | 1.3.0 |
| **Flutter** | 1.1.1 |
| **React Native** | 1.1.1 |
| **Web** | 버전 무관 (ADID 필요) |

지원되지 않는 버전에서는 미리보기 버튼이 비활성화되고 안내 메시지가 표시됩니다.

### 제약사항

| 항목 | 내용 |
|------|------|
| **유효 시간** | 미리보기 설정 후 5분 이내에 확인해야 합니다. |
| **기기 등록** | 미리보기할 기기의 ADID가 미리 등록되어 있어야 합니다. |
| **앱 실행** | 미리보기 설정 후 앱을 새로 실행해야 소재가 노출됩니다. |

## 도착 브라우저 설정

소재 클릭 시 도착 URL이 열리는 방식을 설정할 수 있습니다. 소재 등록 또는 수정 화면에서 설정할 수 있습니다.

### 브라우저 옵션

| 옵션 | 설명 |
|------|------|
| **기본 브라우저** | 기기에 설정된 기본 브라우저에서 열기 (기본값) |
| **인앱 브라우저** | 앱을 벗어나지 않고 앱 내 브라우저에서 열기 |

### 플랫폼별 동작

| 플랫폼 | 기본 브라우저 | 인앱 브라우저 |
|--------|-------------|--------------|
| **iOS** | Safari 등 기본 앱으로 이동 | SFSafariViewController 사용 |
| **Android** | Chrome Custom Tabs 사용 | Chrome Custom Tabs 사용 |
| **Web** | 새 탭에서 열기 | 새 탭에서 열기 |

> 💡 Android는 두 옵션 모두 Chrome Custom Tabs를 사용하므로 동작이 동일합니다. iOS에서 인앱 브라우저를 선택하면 사용자가 앱을 벗어나지 않아 이탈률을 줄일 수 있습니다.

### 웹뷰 환경 주의사항

네이티브 앱이라도 웹뷰로 광고를 표시하는 경우, 인앱 브라우저 동작을 위해 매체사의 추가 구현이 필요합니다.

- **iOS**: SDK 초기화 시 `useInAppBrowser: true` 설정 및 WKNavigationDelegate 구현
- **Android**: WebViewClient에서 `shouldOverrideUrlLoading` 구현 및 Custom Tabs 연동

웹뷰에서 링크 클릭을 가로채어 인앱 브라우저로 열어주는 브릿지 구현이 없으면, 설정과 관계없이 웹뷰 내에서 페이지가 이동하거나 외부 브라우저로 열립니다.

### 딥링크 처리

도착 URL이 앱스킴(예: `kakao://`, `instagram://`)인 경우, 인앱 브라우저 설정과 관계없이 해당 앱으로 직접 이동합니다. 인앱 브라우저는 HTTP/HTTPS URL만 지원합니다.
